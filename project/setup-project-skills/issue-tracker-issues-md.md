# Issue tracker: ISSUES.md

Issues and PRDs for this repo live in a single `ISSUES.md` file at the repo root.

## File format

Each issue is a fenced block separated by `---`. The header line encodes the ID, category, and state:

```markdown
## [ISSUE-001] · enhancement · needs-triage
**Title:** Short title here
**Created:** YYYY-MM-DD

### What to build

Concise description of the vertical slice.

### Acceptance criteria

- [ ] Criterion 1
- [ ] Criterion 2

### Blocked by

None — can start immediately.

---
```

- **ID**: auto-incremented (`ISSUE-001`, `ISSUE-002`, …). Read the file to find the highest existing ID before creating a new one.
- **Category**: `bug` or `enhancement`
- **State**: one of `needs-triage`, `needs-info`, `ready-for-agent`, `ready-for-human`, `wontfix`

## When a skill says "publish to the issue tracker"

Append a new issue block to `ISSUES.md` (creating the file if it doesn't exist). Use the next available ID.

## When a skill says "fetch the relevant ticket"

Read `ISSUES.md` and find the block whose ID or title matches what the user passed.

## When a skill says "apply a label / change state"

Edit the header line of the matching issue in-place:

```
## [ISSUE-001] · enhancement · needs-triage   →   ## [ISSUE-001] · enhancement · ready-for-agent
```

## When a skill says "comment on an issue"

Append a `### Comment` subsection to the matching issue block, before the trailing `---`:

```markdown
### Comment · YYYY-MM-DD

> *This was generated by AI during triage.*

Comment body here.
```

## Listing issues

To show issues by state, read `ISSUES.md` and filter by the state token in the header line. Display as a numbered list with ID, title, and state.
